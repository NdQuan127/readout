# In-Depth Design Critique Guide

> A detailed reference for Phase 7. Provides grading standards, context-specific focus areas, and a checklist of common issues.

---

## Grading Standards in Detail

### 1. Philosophy Alignment

| Score | Criteria |
|------|------|
| 9-10 | The design perfectly embodies the core essence of the chosen philosophy; every detail has a philosophical rationale. |
| 7-8 | The overall direction is correct and core features are in place, with only a few minor details deviating. |
| 5-6 | The intent is visible, but the execution mixes in other stylistic elements, lacking purity. |
| 3-4 | Superficial imitation only, without understanding the philosophical core. |
| 1-2 | Essentially unrelated to the chosen philosophy. |

**Key Review Points**:
- Are the signature techniques of the designer/agency used?
- Do the colors, typography, and layout align with the philosophical framework?
- Are there any "self-contradictory" elements? (e.g., choosing Kenya Hara but cluttering the design with content)

### 2. Visual Hierarchy

| Score | Criteria |
|------|------|
| 9-10 | The user's eye flows naturally along the designer's intended path; information retrieval is frictionless. |
| 7-8 | The primary-secondary relationship is clear, with occasionally 1 or 2 areas of ambiguous hierarchy. |
| 5-6 | Headings and body text can be distinguished, but the intermediate hierarchy levels are muddled. |
| 3-4 | Information is flatly laid out, with no clear visual entry point. |
| 1-2 | Chaotic; the user has no idea where to look first. |

**Key Review Points**:
- Is the font size contrast between the heading and body text sufficient? (at least 2.5x)
- Do the color, weight, and size establish 3–4 clear hierarchical levels?
- Does the whitespace guide the viewer's gaze?
- The "Squint Test": When squinting, is the hierarchy still distinct?

### 3. Craft Quality

| Score | Criteria |
|------|------|
| 9-10 | Pixel-perfect precision; alignment, spacing, and colors are completely flawless. |
| 7-8 | Overall exquisite, with only 1 or 2 minor alignment/spacing issues. |
| 5-6 | Basically aligned, but spacing is inconsistent and the color palette is unsystematic. |
| 3-4 | Obvious alignment errors, chaotic spacing, and too many colors. |
| 1-2 | Rough, looks like a draft. |

**Key Review Points**:
- Is a unified spacing system (such as an 8pt grid) utilized?
- Is spacing between elements of the same type consistent?
- Is the number of colors controlled? (usually no more than 3–4)
- Is the font family unified? (usually no more than 2)
- Are the edge alignments precise?

### 4. Functionality

| Score | Criteria |
|------|------|
| 9-10 | Every design element serves the objective; zero redundancy. |
| 7-8 | Clearly function-oriented, with a small amount of non-essential decoration. |
| 5-6 | Basically usable, but with obvious decorative elements distracting the user. |
| 3-4 | Form over function; the user has to search hard for information. |
| 1-2 | Completely overwhelmed by decoration, losing the ability to convey information. |

**Key Review Points**:
- If you remove any single element, does the design get worse? (If not, it should be deleted.)
- Are the CTA and key information placed in the most prominent position?
- Are there elements added "just because they look good"?
- Does the information density match the medium? (PPTs should not be too dense, whereas PDFs can support higher density.)

### 5. Originality

| Score | Criteria |
|------|------|
| 9-10 | Refreshing and innovative; finds a unique expression within the chosen philosophical framework. |
| 7-8 | Demonstrates independent thought; not just a simple template application. |
| 5-6 | Standard and unremarkable; looks like a template. |
| 3-4 | Heavy use of clichés (e.g., gradient spheres representing AI). |
| 1-2 | Entirely compiled from templates or stock assets. |

**Key Review Points**:
- Are common clichés avoided? (See the "Common Issues Checklist" below)
- Is there personal expression while adhering to the design philosophy?
- Are there "unexpected but reasonable" design decisions?

## Context-Specific Review Priorities

Different output types have different review priorities:

| Context | Highest Priority Dimension | Secondary Priority | Can Be Relaxed |
|------|-----------|--------|--------|
| Official Account Cover/Illustration | Originality, Visual Hierarchy | Philosophy Alignment | Functionality (single image, no interaction) |
| Infographics | Functionality, Visual Hierarchy | Craft Quality | Originality (accuracy takes precedence) |
| PPT/Keynote | Visual Hierarchy, Functionality | Craft Quality | Originality (clarity takes precedence) |
| PDF/Whitepapers | Craft Quality, Functionality | Visual Hierarchy | Originality (professionalism takes precedence) |
| Landing Pages/Official Websites | Functionality, Visual Hierarchy | Originality | — (Comprehensive requirements) |
| App UI | Functionality, Craft Quality | Visual Hierarchy | Philosophy Alignment (usability takes precedence) |
| Xiaohongshu Illustration | Originality, Visual Hierarchy | Philosophy Alignment | Craft Quality (atmosphere takes precedence) |

## Top 10 Common Design Issues

### 1. AI Tech Clichés
**Issue**: Gradient spheres, digital rain, blue circuit boards, robot faces
**Why it's an issue**: Users are visually fatigued by these, making it impossible to differentiate you from others.
**Fix**: Replace literal symbols with abstract metaphors (e.g., using a metaphor for "dialogue" instead of a chat bubble icon).

### 2. Insufficient Font Size Hierarchy
**Issue**: The difference between the heading and body text is too small (<2.5x).
**Why it's an issue**: Users cannot quickly locate key information.
**Fix**: Headings should be at least 3x the size of the body text (e.g., body text 16px → headings 48–64px).

### 3. Too Many Colors
**Issue**: Using 5 or more colors with no clear dominance or hierarchy.
**Why it's an issue**: Causes visual clutter and weakens brand identity.
**Fix**: Limit to 1 primary color + 1 secondary color + 1 accent color + grayscale.

### 4. Inconsistent Spacing
**Issue**: Spacing between elements is arbitrary, lacking a system.
**Why it's an issue**: Looks unprofessional and disrupts the visual rhythm.
**Fix**: Establish an 8pt grid system (only using spacing increments of 8/16/24/32/48/64px).

### 5. Insufficient Whitespace
**Issue**: Every bit of space is crammed with content.
**Why it's an issue**: Information crowding leads to reading fatigue and actually reduces information delivery efficiency.
**Fix**: Whitespace should occupy at least 40% of the total area (60%+ for minimalist styles).

### 6. Too Many Fonts
**Issue**: Using 3 or more fonts.
**Why it's an issue**: Creates visual noise and weakens the sense of unity.
**Fix**: Use a maximum of 2 fonts (1 for headings + 1 for body text), creating variation using font weights and sizes.

### 7. Inconsistent Alignment
**Issue**: Mix of left-aligned, centered, and right-aligned elements.
**Why it's an issue**: Disrupts the sense of visual order.
**Fix**: Select one alignment method (left alignment is recommended) and apply it consistently throughout.

### 8. Decoration Overpowering Content
**Issue**: Background patterns, gradients, or shadows steal the spotlight from the primary content.
**Why it's an issue**: Puts the cart before the horse; users look for information, not decoration.
**Fix**: Ask, "If I remove this decoration, will the design get worse?" If not, delete it.

### 9. Cyber Neon Overuse
**Issue**: Dark blue background (#0D1117) combined with neon glow effects.
**Why it's an issue**: Default aesthetic danger zone (the taste baseline of this skill) and has become one of the biggest clichés. Users can override it according to their own brand.
**Fix**: Choose a more distinctive color scheme (refer to the 20 style color systems).

### 10. Mismatch Between Information Density and Medium
**Issue**: Putting a full page of text in a PPT, or stuffing 10 elements into a cover image.
**Why it's an issue**: Different mediums have different optimal information densities.
**Fix**:
- PPT: 1 core point per slide.
- Cover Image: 1 visual focal point.
- Infographics: Layered presentation.
- PDF: Can be denser, but requires clear navigation.
